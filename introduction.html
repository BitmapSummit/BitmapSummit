PART ONE

What Are Bitcoin Ordinals?

Bitcoin Ordinals represent a groundbreaking innovation launched on Bitcoin mainnet by developer Casey Rodarmor in January 2023. The protocol transformed the Bitcoin blockchain by enabling the storage of diverse data types directly onto individual satoshis.

A satoshi (sat) is the smallest unit of Bitcoin — 1 BTC = 100,000,000 satoshis. The Ordinals protocol assigns a sequential number to each sat based on the order it was mined, creating a permanent identifier. Any data inscribed on that sat becomes a permanent, immutable record on the world's most secure blockchain.

This allows texts, images, videos, audio, and any other digital content to be immortalized on Bitcoin — expanding its utility far beyond financial transactions into digital collectibles and data preservation.

How Ordinals Work

⛏️
1. Mining Order
Each satoshi is numbered in the order it is mined. The first sat of the genesis block is ordinal 0, and numbers increment with every sat ever created.
📝
2. Inscription
Data is "inscribed" onto a specific sat using a Bitcoin transaction. This embeds the content directly in the blockchain — not on external storage like IPFS.
🔒
3. Immutability
Once inscribed, the data is secured by Bitcoin's proof-of-work. It cannot be changed, deleted, or censored — as permanent as Bitcoin itself.
💱
4. Transferability
Inscribed sats can be sent like regular Bitcoin, meaning ownership of the inscription transfers with the sat. No bridge, no wrapped token — pure Bitcoin.
PART TWO

What Is Bitmap?

Bitmap is a consensus standard that allows anybody to claim the geospatial digital real estate of a Bitcoin Block. This process is achieved by being the first to inscribe {block-height}.bitmap unto a Satoshi using the Ordinals Inscription standard. This process is equitable and decentralized, transmuting any Bitcoin Block into a part of the Metaverse at the cost of energy. — Bitmap Gitbook, Bitoshi Blockamoto

Where Ordinals allow inscribing data onto a single satoshi, Bitmap takes the concept further — allowing users to claim ownership of an entire Bitcoin block. By inscribing a specific format (blockheight.bitmap), the first person to do so claims that block as their district.

Each Bitcoin block contains a specific set of transactions, block data, timestamps, and value information. Bitmap Theory maps all of this data into spatial analogues — turning abstract blockchain data into conceptual land, area, and depth in a virtual metaverse.

This makes every Bitmap non-arbitrary — unlike most NFTs generated from randomized traits, Bitmaps are determined entirely by real, on-chain Bitcoin data.

Core Bitmap Concepts

🏙️
District
A District is an entire Bitcoin block claimed via inscription. The block height number is the district ID. Block 0 = the rarest district — Satoshi's genesis block.
🏡
Parcel
Each transaction within a block becomes a Parcel of land. Parcels are children of their District — they transfer when the District is sold. Size is determined by transaction value.
🎨
Blocktribute
Owners can add attributes to their block — art, metadata, creative content. This can increase value and create unique experiences on the block. Only verified owners can do this.
⚡
Non-Arbitrary
Bitmap's key innovation: all data is derived from real Bitcoin block data. No randomized traits. No central authority decides appearance — the blockchain itself does.
🌍
Digital Matter Theory
Bitmap is an application of DMT — the idea that on-chain data has intrinsic matter properties. Each block's data is a unique "digital substance" with real, verifiable characteristics.
🔓
Open Standard
Bitmap is open-source and permissionless. Any team can build indexers, visualizers, or games on top of it. The protocol belongs to no single company — it belongs to Bitcoin.
